# 人名分类器（RNN/LSTM/GRU 实现）
基于 PyTorch 深度学习框架构建的人名所属国家/语种类别分类器，深度对比 RNN、LSTM、GRU 三种经典循环神经网络在序列文本分类任务中的性能表现。本项目完整覆盖数据预处理、模型构建、训练优化、性能可视化、模型预测全流程，可作为自然语言处理领域序列任务的入门实践案例，也可基于此扩展更复杂的文本分类场景。

## 功能
- 支持 18 个国家/语种的人名分类，涵盖 Italian、English、Arabic、Spanish、Scottish、Irish、Chinese、Vietnamese、Japanese、French、Greek、Dutch、Korean、Polish、Portuguese、Russian、Czech、German
- 多维度对比 RNN/LSTM/GRU 模型性能：实时记录并可视化训练过程中的平均损失、准确率变化曲线，统计不同模型的整体训练耗时
- 提供便捷的单人名预测功能：输入任意英文人名，输出 Top3 预测的国家/语种结果，并附带置信度（LogSoftmax 概率值），直观展示模型预测效果
- 模型权重持久化：训练过程中按轮次保存模型参数，支持加载预训练模型快速进行预测，无需重复训练
- 完善的训练监控：每 100 步统计一次平均损失和准确率，每 2000 步打印详细训练日志，便于实时掌握模型训练状态

## 环境准备
### 基础环境要求
- Python 版本：3.7 及以上（兼容 PyTorch 1.8+ 版本）
- 操作系统：Windows/Linux/macOS 均可，无特殊环境依赖

## 模型架构说明
| 模型   | 输入维度 | 隐藏层维度 | 输出维度 | 核心特点                     |
|--------|----------|------------|----------|------------------------------|
| RNN    | 57       | 128        | 18       | 基础循环结构，实现简单但易出现梯度消失 |
| LSTM   | 57       | 128        | 18       | 引入门控机制，缓解梯度消失，擅长捕捉长序列依赖 |
| GRU    | 57       | 128        | 18       | 简化版 LSTM，参数更少，训练效率更高 |

## 注意事项
1. 数据集路径：若修改 `name_classfication.txt` 存放位置，需同步调整代码中 `read_data` 函数的 `filepath` 参数
2. 训练轮数：默认训练 1 轮（快速验证），实际应用建议将 `epochs` 参数调整为 10+，以获得更优的模型效果
3. 模型保存：`save_model` 目录会自动创建，训练完成后模型权重文件命名格式为 `ai23_{model}_{epoch}.bin`
4. 中文可视化：绘图时已适配 SimHei 中文字体，若出现乱码需确保系统已安装对应字体
5. 预测依赖：执行预测函数前，需先完成对应模型的训练，确保 `save_model` 目录下存在该模型的权重文件

## 扩展方向
- 超参数优化：调整学习率、隐藏层维度、批次大小、训练轮数等，进一步提升模型准确率
- 数据增强：对人名数据进行随机大小写转换、字符替换等增强操作，提升模型鲁棒性
- 模型升级：添加 Dropout 层抑制过拟合，或使用双向循环神经网络（Bi-LSTM/Bi-GRU）提升特征提取能力
- 功能扩展：划分训练/测试集评估模型泛化能力，或封装为 API 接口实现快速调用
